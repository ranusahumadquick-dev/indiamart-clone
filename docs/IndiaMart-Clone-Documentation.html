What this project is and what it accomplishes

IndiaMart Clone is a comprehensive full-stack B2B (Business-to-Business) marketplace platform inspired by India's largest online B2B directory. Problem Statement
The real-world challenge this platform addresses

Small and medium-sized Indian enterprises face critical barriers in B2B commerce. Traditional offline procurement is slow, geographically limited, and opaque. This platform solves six core problems:

🔍

Discovery Gap

Buyers struggle to find verified quality suppliers across India's vast geography.

💬

Communication Barrier

No structured channel for buyers to send inquiries and receive quotes from multiple suppliers.

✅

Trust Deficit

Buyers cannot verify supplier credibility or certifications without a physical visit.

📊

Seller Invisibility

Small manufacturers have no digital presence to reach a pan-India buyer base.

💳

Payment Friction

No integrated Indian payment system for subscription upgrades and premium features.

📱

Mobile Access

Existing solutions lack mobile-first responsive design for India's smartphone market.

Solution: This platform provides verified seller profiles, structured inquiry management, RFQ board, product comparison, subscription plans, and Razorpay integration — all in a mobile-first responsive interface.

Authentication System
JWT-based auth with role-based access control
1

Registration

User submits name, email, password, role. Mongoose pre-save hook hashes password with bcrypt (10 salt rounds). User saved to MongoDB.

2

Login

Email + password verified. Two JWTs issued: accessToken (1 day) and refreshToken (7 days). Refresh token stored in DB.

3

Route Protection

authMiddleware extracts Bearer token from Authorization header, verifies with JWT_SECRET, attaches user to req.user.

4

Role Authorization

roleMiddleware('seller','admin') checks req.user.role against allowed roles. Returns 403 if unauthorized.

5

Token Refresh

Axios interceptor catches 401 → calls /api/auth/refresh → gets new access token → retries original request transparently.

📧
Email + Password Login

JWT access token (1d) issued on success; Axios interceptor auto-refreshes on 401

👤
Role-Based Registration

Users choose Buyer or Seller at signup — role stored in MongoDB, controls dashboard access

🔐
bcrypt Password Hashing

Mongoose pre-save hook hashes passwords at 10 salt rounds before every write

🔄
Refresh Token Rotation

7-day refresh token stored in DB; cleared on logout; new access token silently issued

RFQ System (Buy Requirements)
Request for Quotation — buyers post requirements, sellers respond

The RFQ (Request for Quotation) board allows buyers to post specific product requirements with quantity, target price, and deadline. Verified sellers can browse these requirements and contact interested buyers directly.

1

Buyer Posts Requirement

Buyer fills title, description, category, quantity, target price, and expiry date. Posted publicly on the RFQ board.

2

Sellers Browse RFQ Board

Any verified seller can see all active buy requirements filtered by category, quantity, and date.

3

Seller Contacts Buyer

Interested seller sends an inquiry with their quote. Buyer receives notification and can compare quotes.

4

Requirement Expires

RFQs automatically expire on the set date. Buyers can renew or mark as fulfilled.

📋
Public RFQ Board

All active buy requirements visible to verified sellers — filterable by category and date

📅
Expiry Dates

💰
Target Price & Quantity

Buyers specify quantity, unit, target price — sellers can evaluate fit before contacting

🔔
Seller Notifications

Sellers receive in-app notification when a new RFQ matches their product category

Security Implementation
All security layers applied in this project
🔒

Password Security

bcrypt with 10 salt rounds. Mongoose pre-save hook hashes before every save automatically.

🎫

JWT Auth

Short-lived access tokens (1d) + long-lived refresh tokens (7d). Refresh tokens stored in DB, cleared on logout.

🛡️

Helmet.js

Sets 14 HTTP security headers: CSP, X-Frame-Options, Strict-Transport-Security, and more.

⏱️

Rate Limiting

100 req / 15 min per IP. Auth routes: stricter at 10 req / 15 min to prevent brute-force attacks.

🌐

CORS Policy

Only allows CLIENT_URL. Credentials enabled for httpOnly cookie-based refresh tokens.

✅

Payment Security

Razorpay HMAC-SHA256 signature verification. Payments cannot be faked without the key secret.

Role Authorization: Every protected route has both authMiddleware (token check) AND roleMiddleware (role check). A buyer with valid token still cannot access seller-only routes.

Deployment Guide
Local setup and production deployment
1

Install all dependencies

npm run install:all from project root installs both frontend and backend packages.

2

Configure backend environment

Create backend/.env with MongoDB URI, JWT secrets, Cloudinary keys, Razorpay keys, CLIENT_URL.

3

Configure frontend environment

Create frontend/.env.local with NEXT_PUBLIC_API_URL=http://localhost:8000/api

4

Seed the database

cd backend && node src/seed/seed.js — creates categories, subscription plans, and test users.

5

Start development

npm run dev — starts frontend (:3000) and backend (:8000) concurrently.

# backend/.env
PORT=8000
MONGO_URI=mongodb://127.0.0.1:27017/indiamart
JWT_SECRET=your-jwt-secret-min-32-chars
JWT_ACCESS_EXPIRY=1d
JWT_REFRESH_SECRET=your-refresh-secret
JWT_REFRESH_EXPIRY=7d
