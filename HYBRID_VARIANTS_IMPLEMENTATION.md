# 🎯 Hybrid Variant Management System - Complete Implementation

## Overview

This document describes the complete **hybrid variant management system** for the IndiaMART B2B marketplace, combining automatic variant generation from categories with manual variant management in the Seller Panel.

## System Architecture

### Phase 1: Auto-Generation (Existing)
- **Category-based templates**: Variants auto-generate from `AUTO_VARIANTS` mapping when creating/editing products
- **13+ categories** with subcategory-specific variant templates
- **Cartesian product** generation: combines all attribute values into unique variants
- **Stock distribution**: divides product stock equally among variants

### Phase 2: Manual Management (NEW)
- **Seller Panel interface** for complete variant CRUD
- **Variant table** with search, filters, pagination
- **Bulk operations**: Update price, stock, status for multiple variants
- **Individual editing**: Modify price, stock, status per variant
- **Regeneration control**: Preserve manual variants while regenerating auto-variants

### Hybrid Approach
- Auto-variants generated by default on category selection
- Sellers can customize variants anytime after creation
- Manual variants coexist with auto-generated ones
- Status tracking identifies variant source (auto vs manual)

## Backend API Endpoints

### Base URL
All endpoints use the pattern: `/api/products/:productId/variants/`

### Endpoints

#### 1. List Variants (Paginated with Filters)
```
GET /api/products/:productId/variants/list

Query Parameters:
  - page (default: 1)
  - limit (default: 20)
  - search: SKU search term
  - status: Filter by "active|inactive|out_of_stock"
  - stockFilter: "in_stock|low_stock|out_of_stock"
  - sortBy: "sku|price|stock|status|created"

Response:
{
  success: true,
  data: {
    variants: [...],
    totalCount: 100,
    page: 1,
    limit: 20,
    hasMore: true,
    totalPages: 5
  }
}
```

#### 2. Create Variant
```
POST /api/products/:productId/variants/create

Body:
{
  sku: "PROD-RED-S-1",
  attributeValues: { "Color": "Red", "Size": "Small" },
  price: 599,
  stock: 50,
  status: "active"  // optional
}

Auth: Requires seller role
```

#### 3. Update Variant
```
PUT /api/products/:productId/variants/update/:variantId

Body:
{
  price: 599,          // optional
  stock: 50,           // optional
  status: "active"     // optional
}

Auth: Requires seller role
```

#### 4. Delete Variant
```
DELETE /api/products/:productId/variants/delete/:variantId

Auth: Requires seller role
```

#### 5. Bulk Update
```
PUT /api/products/:productId/variants/bulk-update

Body:
{
  action: "price|stock|status|activate|deactivate|mark_out_of_stock",
  variantIds: ["id1", "id2", "id3"],
  value: 599  // for price/stock/status actions
}

Response:
{
  success: true,
  data: {
    updated: 3
  }
}

Auth: Requires seller role
```

#### 6. Regenerate Auto-Variants
```
POST /api/products/:productId/variants/regenerate

Body:
{
  preserveManualVariants: true  // Keep existing manual variants
}

Response:
{
  success: true,
  data: {
    autoVariants: 12,
    manualVariants: 3,
    totalVariants: 15
  }
}

Auth: Requires seller role
```

#### 7. Search Variants
```
GET /api/products/:productId/variants/search

Query:
  - q: SKU search term
  - limit: Max results (default: 10)

Response: Array of matching variants
```

## Frontend Components

### Location
All components are in: `/frontend/src/components/seller/variants/`

### Components

#### 1. **VariantManagementTable**
Displays variant data in table format.

**Features:**
- Sortable columns: SKU, Price, Stock, Status
- Selectable rows with checkbox select-all
- Status badges (Active, Inactive, Out of Stock)
- Source badges (Auto, Manual)
- Inline action buttons: Edit, Delete

**Props:**
```typescript
interface VariantManagementTableProps {
  variants: Variant[];
  loading: boolean;
  selectedIds: string[];
  onSelectVariant: (id: string) => void;
  onSelectAll: () => void;
  onEdit: (variant: Variant) => void;
  onDelete: (variantId: string) => void;
  isAllSelected: boolean;
}
```

#### 2. **VariantSearchFilter**
Search and filter controls.

**Features:**
- SKU search (real-time)
- Status filter
- Stock level filter
- Sort selector
- Reset button

#### 3. **VariantEditModal**
Modal for editing variant price, stock, and status.

**Features:**
- Read-only SKU and variant combination
- Editable price, stock, status
- Form validation
- Submit/Cancel buttons

#### 4. **BulkActionsPanel**
Bulk operations interface.

**Features:**
- Collapsible panel (shows count of selected variants)
- Action selector
- Value input (dynamic based on action)
- Apply button
- Actions: Update Price, Update Stock, Activate All, Deactivate All, Mark Out of Stock

#### 5. **PaginationControl**
Navigation for large variant lists.

**Features:**
- Previous/Next buttons
- Page number buttons
- Current/Total page display
- Results counter

### Custom Hooks

#### **useVariantManagement**
Location: `/frontend/src/hooks/useVariantManagement.ts`

```typescript
const management = useVariantManagement({ productId, pageSize: 20 });

// Properties
management.variants          // Array of variants
management.loading           // Loading state
management.error             // Error message
management.filters           // Current filters
management.pagination        // Pagination state
management.selectedVariantIds // Selected variant IDs

// Methods
management.fetchVariants()   // Fetch with current filters
management.createVariant(data)
management.updateVariant(variantId, data)
management.deleteVariant(variantId)
management.bulkUpdate(action, value)
management.regenerateVariants(preserveManual)
management.applyFilters(filterObj)
management.toggleVariantSelection(id)
management.toggleSelectAll()
management.goToPage(pageNum)
```

## Performance Considerations

1. **Pagination**: Max 20 variants per page by default (prevents slow loads)
2. **Indexes**: Database queries optimized with variant SKU and status indexes
3. **Search**: SKU search is case-insensitive and fast
4. **Bulk Operations**: MongoDB batch updates (single query for multiple records)
5. **Lazy Loading**: Modals load only when opened

## Support & Troubleshooting

### Issue: Variants not showing in table
**Solution:** Check filters - reset all filters to see all variants

### Issue: Bulk actions disabled
**Solution:** Select at least one variant using checkboxes

### Issue: Cannot delete auto-variant
**Solution:** Regenerate variants to reset, or edit individual variants

### Issue: Price/Stock not updating
**Solution:** Ensure modal form is submitted (check for validation errors)

### Issue: Pagination not working
**Solution:** Ensure `pageSize` prop is set correctly in hook (default: 20)
